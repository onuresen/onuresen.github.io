# Starting a notes section

This site used to be just a list of projects. That's fine, but it doesn't show how any of them got built, and it never updates between releases.

So: a notes section. Short, low-stakes posts — a paragraph on something I learned, a decision I made on a project, a tool I tried. Not a blog with a publishing schedule, just a log.

It's markdown files in a folder, plus a small JSON manifest. No build step, no CMS — write a `.md` file, add one line to `notes/index.json`, commit, push. Same philosophy as the rest of this site: plain HTML, zero dependencies, GitHub Pages.

More to come as projects move.
